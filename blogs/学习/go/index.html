    # Go语言起源

    编程语言的演化跟生物物种的演化类似，一个成功的编程语言的后代一般都会继承它们祖先的优点；当然有时多种语言杂合也可能会产生令人惊讶的特性；还有一些激进的新特性可能并没有先例。通过观察这些影响，我们可以学到为什么一门语言是这样子的，它已经适应了怎样的环境。

    下图展示了有哪些早期的编程语言对Go语言的设计产生了重要影响。

    Go语言有时候被描述为“C类似语言”，或者是“21世纪的C语言”。Go从C语言继承了相似的表达式语法、控制流结构、基础数据类型、调用参数传值、指针等很多思想，还有C语言一直所看中的编译后机器码的运行效率以及和现有操作系统的无缝适配。

    但是在Go语言的家族树中还有其它的祖先。其中一个有影响力的分支来自Niklaus Wirth (opens new window)所设计的Pascal (opens new window)语言。然后Modula-2 (opens new window)语言激发了包的概念。然后Oberon (opens new window)语言摒弃了模块接口文件和模块实现文件之间的区别。第二代的Oberon-2 (opens new window)语言直接影响了包的导入和声明的语法，还有Oberon (opens new window)语言的面向对象特性所提供的方法的声明语法等。

    Go语言的另一支祖先，带来了Go语言区别其他语言的重要特性，灵感来自于贝尔实验室的Tony Hoare (opens new window)于1978年发表的鲜为外界所知的关于并发研究的基础文献 顺序通信进程 （ communicating sequential processes (opens new window) ，缩写为CSP (opens new window)。在CSP (opens new window)中，程序是一组中间没有共享状态的平行运行的处理过程，它们之间使用管道进行通信和控制同步。不过Tony Hoare (opens new window)的CSP (opens new window)只是一个用于描述并发性基本概念的描述语言，并不是一个可以编写可执行程序的通用编程语言。

    接下来，Rob Pike和其他人开始不断尝试将CSP (opens new window)引入实际的编程语言中。他们第一次尝试引入CSP (opens new window)特性的编程语言叫Squeak (opens new window)（老鼠间交流的语言），是一个提供鼠标和键盘事件处理的编程语言，它的管道是静态创建的。然后是改进版的Newsqueak (opens new window)语言，提供了类似C语言语句和表达式的语法和类似Pascal (opens new window)语言的推导语法。Newsqueak是一个带垃圾回收的纯函数式语言，它再次针对键盘、鼠标和窗口事件管理。但是在Newsqueak语言中管道是动态创建的，属于第一类值，可以保存到变量中。

    在Plan9操作系统中，这些优秀的想法被吸收到了一个叫Alef (opens new window)的编程语言中。Alef试图将Newsqueak语言改造为系统编程语言，但是因为缺少垃圾回收机制而导致并发编程很痛苦。（译注：在Alef之后还有一个叫Limbo (opens new window)的编程语言，Go语言从其中借鉴了很多特性。 具体请参考Pike的讲稿：http://talks.golang.org/2012/concurrency.slide#9 ）

    Go语言的其他的一些特性零散地来自于其他一些编程语言；比如iota语法是从APL (opens new window)语言借鉴，词法作用域与嵌套函数来自于Scheme (opens new window)语言（和其他很多语言）。当然，我们也可以从Go中发现很多创新的设计。比如Go语言的切片为动态数组提供了有效的随机存取的性能，这可能会让人联想到链表的底层的共享机制。还有Go语言新发明的defer语句。
